## 11.1 評価順序

**[規範] RUN-001 (MUST)**
式評価は規則で定義された順序で決定的に行う。

**[規範] RUN-002 (MUST)**
同一入力・同一外部状態では同一結果を返す。

---

## 11.2 非同期予約語

**[規範] RUN-003 (MUST)**
`しながら`、`待つ`、`背景で` の評価は常に未実装エラーとする。

---

## 11.3 標準 I/O API

**[規範] RUN-004 (MUST)**
`表示する` は `表示する【内容:を】` として解釈し、`内容` を表示したうえで `無` を返す。

**[規範] RUN-005 (MUST)**
`入力する` は 0 引数の標準手順として解釈し、標準入力から 1 行を読み込んだ文字列を返す。

**[規範] RUN-006 (MUST)**
`「ファイル」を 読む` は `文字列` を返す標準手順として解釈し、読み込み失敗時は実行時エラーとする。

**[規範] RUN-007 (MUST)**
`「内容」を 「ファイル」に 書く` は `無` を返す標準手順として解釈し、書き込み失敗時は実行時エラーとする。

**[規範] RUN-008 (MUST NOT)**
`入力を受ける` は v1 の標準 I/O API 名として扱ってはならない。入力 API 名は `入力する` に統一する。
