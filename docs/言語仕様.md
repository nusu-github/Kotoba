# 言（ことば）言語仕様 v0.4

> 日本語の思考構造から設計されたプログラミング言語

---

## 設計思想

プログラミング言語の多くは英語を母体としている。`if`、`for`、`return`——これらは英語話者にとっては意味のある単語だが、非英語話者にとっては最初に丸暗記すべき記号にすぎない。このこと自体に悪意はないが、結果として世界の大半のプログラマーは「英語で考えてからコードを書く」という翻訳コストを無意識に支払い続けている。

**言（ことば）** は、日本語話者が日本語のまま考え、日本語のまま書けるプログラミング言語である。

ただし、単にキーワードを日本語に置き換えた言語ではない。日本語には日本語の文法がある。助詞が関係を示し、語順が自由で、主語は省略され、動詞が最後に来る。言はこの構造そのものを構文に取り込む。

### 曖昧さの哲学

日本語の本質的な特徴のひとつに曖昧さがある。主語の省略、「それ」の多義性、文脈依存の解釈——これらは日常会話を豊かにするが、プログラミング言語にとっては本来相容れないものである。

言は、この緊張に正面から向き合う。

- **書き手の体験としては、日本語の自然さを保つ。** 省略は許され、語順は自由で、指示語が文脈を引き継ぐ。コードを読んだとき、それが日本語の文章として違和感なく読めることを目指す。
- **機械の解釈としては、一切の曖昧さを残さない。** 助詞が意味役割を厳密に決定し、こそあどのスコープ規則が指示対象を一意に解決し、省略の補完規則が形式的に定義される。

つまり、言は**人間にとっての曖昧さと、機械にとっての厳密さを両立させる言語**である。書き手が感じる「日本語らしい自由さ」の裏側で、仕様が曖昧さを確定的に解消する。

日本語でプログラミングできる言語ではなく、**日本語でプログラミングする言語**を目指す。

---

## 0. 設計原則

1. **助詞が構文を決める** — 語順ではなく助詞（を、が、に、で、から、まで）が引数の役割を示す
2. **こそあどがスコープ** — 指示語の距離感がスコープ階層に対応する
3. **て形がパイプ** — 「〜して、〜して、〜する」が処理の連鎖を表す
4. **動詞が最後** — SOV語順。操作（動詞）がブロックや文の末尾に来る
5. **省略は美徳** — 文脈から推論できるものは省略を許す
6. **助数詞は型** — 「3回」「5秒」「2個」は単なる数値ではなく型付きの量
7. **見かけの曖昧、裏の厳密** — 書き手には日本語の自然さを、機械には確定的な解釈を

## 1. 字句構造

### 1.1 文の区切り

- 改行が文の終端
- 一行に複数の文を書く場合は `。` で区切る
- `…` で次の行に継続

**`、` の用途判定規則：**
`、` はて形連鎖の内部区切りとしてのみ使用する。文の区切りには `。` を用いる。

```
「data.csv」を 読んで、行に分けて、表示する    ※ て形連鎖内の「、」
xを 表示する。yを 表示する                      ※ 文区切りの「。」
```

### 1.2 ブロック

インデント（タブまたはスペース）でブロックを表現する。

```
もし xが 3より大きい ならば
　「大きい」と 表示する
そうでなければ
　「小さい」と 表示する
```

### 1.3 コメント

```
※ これは行コメント

（ これは
　 複数行コメント ）
```

### 1.4 文字列リテラル

```
「こんにちは世界」      ※ 基本の文字列
「名前は【名前】です」  ※ 式展開（【】内が評価される）
```

### 1.5 数値リテラル

```
42          ※ 整数
3.14        ※ 小数
１２３      ※ 全角数字も可
```

## 2. 型システム

### 2.1 基本型

| 型名   | 説明                         | リテラル例                    |
| ------ | ---------------------------- | ----------------------------- |
| 数     | 整数と小数（内部で自動判別） | `42`, `3.14`                  |
| 文字列 | テキスト                     | `「hello」`                   |
| 真偽   | 真または偽                   | `真`, `偽`                    |
| 一覧   | 順序付きコレクション         | `【1, 2, 3】`                 |
| 対応表 | キーと値の組                 | `｛名前:「太郎」、年齢: 20｝` |
| 手順   | 第一級関数                   | （後述）                      |
| 無     | 値が存在しない               | `どれでもない`                |

### 2.2 助数詞型（単位付き数値）

助数詞は数値にセマンティックな型を付与する。

```
3回      → 回数型
5秒      → 時間型（秒）
2個      → 個数型
100行    → 行数型
```

助数詞が異なる値同士の暗黙変換は禁止。

```
3回 と 5秒 を足す   → ※ 型エラー：回数と時間は足せない
3秒 と 5秒 を足す   → 8秒（OK）
```

定義可能な助数詞：

```
「ピクセル」という単位を定義する【記号: px】
幅 は 100ピクセル
```

## 3. 変数と束縛

### 3.1 束縛（代入）

`は` で名前に値を束縛する。

```
名前 は 「太郎」
年齢 は 25
一覧 は 【1, 2, 3】
```

### 3.2 再束縛

`を〜に変える` で再代入する。不変がデフォルト。

```
名前 は 「太郎」
名前を 「次郎」に 変える     ※ 再束縛（明示的）
```

可変にしたい場合は宣言時に明示：

```
変わる 数 は 0
数を 数と1の和 に変える
```

### 3.3 こそあどスコープ参照

| 指示語 | スコープ                       | 意味                               |
| ------ | ------------------------------ | ---------------------------------- |
| `これ` | 現在のブロック                 | 直近の結果、またはローカル変数     |
| `それ` | 親ブロック（一つ外）           | 親スコープの環境への参照           |
| `あれ` | 大域（モジュール／グローバル） | 最も外側のスコープの値             |

`それ` は親スコープの**環境オブジェクト**（environment object）への参照である。`それの変数名` で親スコープの変数にアクセスし、`それ` 単独では親スコープの文脈値（直近の式の値）として評価される。

```
設定 は 「本番」                   ※ 大域スコープ

処理A という手順
　結果 は 42

　処理B という手順
　　これ                          ※ → 処理Bのローカル（直近の結果）
　　それの結果                    ※ → 処理Aの「結果」= 42（名前付きアクセス）
　　あれ                          ※ → 大域の「設定」=「本番」
```

**深いネストの場合：名前で捕獲する**

```
処理A という手順
　外の値 は 100

　処理B という手順
　　処理C という手順
　　　それの外の値   ※ → 処理Bのスコープを経由
　　　外の値         ※ → 名前で処理Aのスコープに到達
```

2段以上の遡りは名前参照を強制することで曖昧さを排除する。`それ` は常に1段上のスコープのみを指す。

### 3.4 こそあど系列の拡張

| 指示語 | 意味                                   |
| ------ | -------------------------------------- |
| `ここ` | 現在の実行位置（デバッグ・メタ情報）   |
| `そこ` | 呼び出し元の位置                       |
| `こう` | 現在のブロックの手順参照（再帰に使用） |

```
階乗 という手順【n:を】
　もし nが 1以下 ならば
　　1を 返す
　そうでなければ
　　nと (n から 1を引いたものを こう する) の積を 返す
```

`こう` = 自分自身を指す → 無名再帰が可能。

## 4. 助詞と式

### 4.1 助詞の役割と分類

本言語で使用する助詞は、文法上の機能に応じて**3つのカテゴリ**に分類される。これらは字句上は似ているが、意味カテゴリは明確に異なる。

#### 4.1.1 役割助詞（引数マーカー）

手順呼び出し時に引数の意味役割（semantic role）を決定する。オーバーロード照合の対象となる。

| 助詞   | 役割               | 英語的対応            |
| ------ | ------------------ | --------------------- |
| `を`   | 対象（目的語）     | direct object         |
| `が`   | 主語・主格         | subject / nominative  |
| `に`   | 目標・宛先         | target / destination  |
| `で`   | 手段・方法         | instrument / method   |
| `から` | 起点               | from / source         |
| `まで` | 終点               | to / until            |
| `と`   | 共同・並列         | with / and            |
| `へ`   | 方向               | toward                |
| `より` | 比較基準           | than / compared to    |

#### 4.1.2 アクセス助詞

| 助詞 | 役割               | 英語的対応            |
| ---- | ------------------ | --------------------- |
| `の` | 所属・属性アクセス | possessive / property |

`の` は属性アクセス（`人の名前`）およびスコープ参照（`それの数`）に使われる。手順の引数宣言には使用できず、オーバーロード照合の対象にもならない。

#### 4.1.3 束縛マーカー

| 助詞 | 役割               | 英語的対応            |
| ---- | ------------------ | --------------------- |
| `は` | 束縛（代入）       | binding / assignment  |

`は` は変数束縛の専用マーカーであり、助詞式（`式 助詞`）の一部としては機能しない。オーバーロード照合の対象にもならない。

#### 4.1.4 `が` の用法詳細

`が` は主に条件文・比較文で主語を明示する。

```
もし xが 3より大きい ならば       ※ 主語の明示
リストの長さが 0と等しい          ※ 属性を主語化
```

`は` と `が` の使い分け：`は` は束縛（代入）の専用マーカー。`が` は式中の主語マーカー。

**設計原則**: オーバーロード照合で使用されるのは**役割助詞のみ**である。`の` や `は` を助詞パターンに含めることはできない。

### 4.2 語順自由

助詞があるため、語順を入れ替えても意味が同じ。

```
「hello.txt」を 「出力」に 複製する
「出力」に 「hello.txt」を 複製する
```

パーサは助詞を手がかりに引数を割り当てる。

### 4.3 属性アクセス（の）

```
人の名前              ※ → person.name
リストの長さ          ※ → list.length
リストの3番目         ※ → list[2]（1始まり）
```

### 4.4 算術・比較

**算術糖衣構文（固定パターン）：**

以下の算術表現は、パーサが認識する**固定の糖衣構文パターン**である。`と` や `の` の一般規則ではなく、「和・差・積」という特定の語と組み合わせて初めて算術になる。

```
※ 算術糖衣構文（パターン： aとbの{2}）
aと bの和              ※ a + b
aと bの差              ※ a - b
aと bの積              ※ a * b

※ 動詞形式の算術（一般の助詞式）
aを bで割る            ※ a / b
aを bで割った余り       ※ a % b
```

**糖衣構文の展開規則：**
- `aとbの和` → `和(a, b)` → `a + b`。左側のオペランドが第1引数、右側が第2引数
- 認識される算術語は `和`・`差`・`積` の3語のみ（将来拡張可能）
- `aとb` の `と` はこのパターン内では共同助詞として解釈され、その直後の `の` は属性アクセスではなく構文の一部
- このパターンに該当しない `の` は通常の属性アクセスとして解釈される

**注意**: 今後「`aとbのX`」形式の算術構文を追加する場合は、パーサへの焦き込みではなく、名前解決・降下変換（lowering）フェーズでの拡張を推奨する。

```
※ 比較
aが bより大きい         ※ a > b
aが bより小さい         ※ a < b
aが bと等しい           ※ a == b
aが bと等しくない       ※ a != b
aが b以上              ※ a >= b
aが b以下              ※ a <= b
```

### 4.5 論理演算

```
aが真 かつ bが真        ※ a && b
aが真 または bが真      ※ a || b
aが真 でない            ※ !a
```

## 5. 制御構造

### 5.1 条件分岐

**基本形：**

```
もし 条件 ならば
　処理
そうでなければ
　処理
```

**連鎖：**

```
もし xが 0と等しい ならば
　「ゼロ」と 表示する
もしくは xが 0より大きい ならば
　「正」と 表示する
そうでなければ
　「負」と 表示する
```

### 5.2 パターンマッチ（どれかを調べる）

```
値は どれかを調べる
　1の場合
　　「一」と 表示する
　2の場合
　　「二」と 表示する
　【名前, 年齢】の場合
　　名前を 表示する
　どれでもない場合
　　「不明」と 表示する
```

ワイルドカード：

```
【どれか, 年齢, どれか】の場合    ※ → [_, age, _]
```

### 5.3 繰り返し

**回数指定：**

```
5回 繰り返す
　「hello」と 表示する

5回 繰り返す【i】               ※ iは1から5
　iを 表示する
```

**範囲指定：**

```
1から 10まで 繰り返す【i】
　iを 表示する
```

**条件指定：**

```
xが 10より小さい 間 繰り返す
　xを xと1の和 に変える
```

**一覧の走査：**

```
リストの それぞれについて【要素】
　要素を 表示する
```

**制御：**

```
次へ           ※ continue
抜ける         ※ break
```

## 6. 手順（関数）

### 6.1 手順の定義

```
挨拶する という手順【名前:を】
　「こんにちは、【名前】さん」と 表示する
```

引数宣言は `名前:助詞` の形式で、名前と助詞を `:` で区切る。

```
※ 複数の助詞引数
複製する という手順【元:を、先:に】
　※ 実装...
```

呼び出し時は助詞が自然に対応する：

```
「hello.txt」を 「出力」に 複製する
「出力」に 「hello.txt」を 複製する   ※ 語順入れ替えもOK
```

さらに複雑な例：

```
送る という手順【内容:を、宛先:に、方法:で】
　※ 「内容」「宛先」「方法」が変数名
　※ 助詞「を」「に」「で」が呼び出し時のマッチに使われる
```

呼び出し：

```
「こんにちは」を 「太郎」に メールで 送る
```

**短縮形（引数名と助詞が自明な場合）：**

引数名を省略して助詞のみでも宣言可能。その場合、助詞がそのまま引数名になる。

```
表示する という手順【:を】
　※ 「を」が引数名として使われる
```

**助詞の一意性制約：**

一つの手順定義内で、同じ助詞を複数の引数に使うことはできない。語順自由の保証は、各助詞が一意に引数を特定できることに依存する。

```
※ NG: 同じ助詞「を」が2回
移す という手順【元:を、先:を】      ※ コンパイルエラー

※ OK: 異なる助詞で役割を分ける
移す という手順【元:を、先:に】      ※ 「を」と「に」で一意
```

### 6.2 戻り値

```
二乗する という手順【x:を】
　xとxの積を 返す

結果 は 5を 二乗する      ※ → 25
```

明示的な `返す` がない場合、最後の式の値が戻り値。

**`返す` の位置付け：**

`返す` は**文キーワード**であり、動詞（手順名）としては解決されない。

- `xを 返す` → `Return(x)` — 正しい使用
- `返す x` → 構文エラー（`返す` の前に `を` 引数が必要）
- `返す` 単独 → `Return(無)` — 値なしの戻り
- `返す` はオーバーロード照合の対象に**ならない**。ユーザーが `返す` という名前の手順を定義することはできない

### 6.3 手順は第一級

```
処理 は （【x:を】xとxの積を返す）   ※ 無名手順

リストを 処理で それぞれ変換する
```

### 6.4 クロージャ

手順は定義時のスコープを捕獲する。`それ` による親スコープ参照でキャプチャした変数にアクセスする。

```
カウンター という手順
　変わる 数 は 0
　増やす という手順
　　それの数を それの数と1の和 に変える   ※ 「それの数」= 親スコープの変数「数」
　　それの数を 返す
　増やすを 返す

数える は カウンター
数えるを実行する    ※ → 1
数えるを実行する    ※ → 2
```

**クロージャにおける `それ` の解釈：**

- `それの数` は「親スコープ（`カウンター`）の変数 `数`」へのアクセスである
- `それ` はクロージャの**定義時**の親スコープを参照する（呼び出し時ではない）
- `変わる` で宣言された変数は、`それの変数名` を通じて変更可能

**名前による直接キャプチャも可能：**

```
カウンター という手順
　変わる 数 は 0
　増やす という手順
　　数を 数と1の和 に変える   ※ 名前で直接参照（クロージャキャプチャ）
　　数を 返す
　増やすを 返す
```

`それの変数名` と名前直接参照は等価である。明示性を重視する場合は `それの` 、簡潔さを重視する場合は名前直接参照を使う。

### 6.5 再帰（こう）

`こう` は現在の手順自身を指す。

```
フィボナッチ という手順【n:を】
　もし nが 1以下 ならば
　　nを 返す
　そうでなければ
　　(nから1を引いたものを こうする) と (nから2を引いたものを こうする) の和を 返す
```

## 7. て形チェイニング（パイプライン）

`て形` で手順を連鎖する。前の結果が次の `を` 引数に暗黙的に渡る。

```
「data.csv」を 読んで、行に分けて、それぞれ表示する
```

これは以下と等価：

```
一時 は 「data.csv」を 読む
一時 は 一時を 行に分ける
一時の それぞれについて【行】
　行を 表示する
```

### 7.1 て形の文法規則

- 動詞の**て形**（読んで、分けて、足して…）が連鎖のシグナル
- 各ステップの戻り値が次のステップの `を` 引数になる
- 最後の動詞は**終止形**（表示する、返す…）

### 7.2 途中分岐（分岐して）

て形チェインの途中で分岐したい場合、`分岐して` を使う。分岐は値を返し、チェインが継続する。

```
「data.csv」を 読んで、行に分けて、分岐して
　もし これの長さが 0と等しい ならば
　　【「（空）」】を 返す
　そうでなければ
　　これを 返す
それぞれ表示する
```

`分岐して` の規則：
- 分岐ブロックは必ず値を `返す` こと（チェインを継続するため）
- 分岐の結果が次のステップの `を` 引数になる
- ブロック終了後、インデントを戻してチェインを再開する

**分岐しない単純な条件付き中断は従来通り：**

```
「data.csv」を 読んで、行に分けて
もし これの長さが 0と等しい ならば
　「空ファイル」と 表示する
```

## 10. エラー処理

```
試す
　「data.csv」を 読む
失敗した場合【問題:で】
　「読み込み失敗：【問題】」と 表示する
必ず行う
　「処理完了」と 表示する
```

意図的にエラーを起こす：

```
「不正な値です」と 訴える       ※ throw相当
```

### 10.1 `試す` の値規則

`試す` は式として値を返す。その規則は以下の通り：

| ブロック | 値 |
|--------|-----|
| `試す` 本体（成功時） | 本体の最後の式の値 |
| `失敗した場合` | そのブロックの最後の式の値 |
| `必ず行う` | 値は**常に破棄**される（副作用のみ） |

**詳細規則：**

- `試す` 全体の値は、成功した場合は本体の値、失敗した場合は `失敗した場合` ブロックの値
- `必ず行う` ブロックの値は常に破棄される。`試す` 全体の値に影響しない
- `必ず行う` 内で `訴える` を実行した場合、元の例外は**上書きされる**（新しい例外が伝搬する）。これは危険なパターンであり、警告を発する
- `失敗した場合` がない `試す` は、例外発生時に値が `無` となる

## 15. コンパイルフェーズと曖昧さの解消

この言語は「曖昧に書けるが、機械は確定的に解釈する」を芯に持つ。曖昧さの解消はコンパイルの各段階で段階的に行われる。

### 15.1 字句解析（レキサー）で決めるもの

| 責務 | 規則 |
|------|------|
| トークン分割 | 空白・記号・括弧でトークンを区切る。識別子の末尾に助詞が付着している場合（`名前を`）は一つのトークンとして読み取り、後段で `名前` + `を` に分離する |
| キーワード優先 | キーワード値完全一致が助詞分離より**常に優先**。`どれかを調べる` は単一キーワード、`そうでなければ` も分割しない |
| 助詞の分離 | 識別子末尾からの**最長一致**で分離。分離対象は**役割助詞**（`を/が/に/で/から/まで/と/へ/より`）、**アクセス助詞**（`の`）、**束縛マーカー**（`は`）の全カテゴリ。トークン種別はカテゴリごとに区別する |
| 動詞の判定 | レキサーでは**行わない**。動詞は識別子として読み取り、パーサに委ねる |
| 数値＋助数詞 | 数値直後の**漢字・カタカナ**列を助数詞候補トークンとして分離。**ひらがなは助数詞文字に含めない**（`2の` の `の` が助数詞に吸われるのを防止） |
| 空白の扱い | 空白はトークン区切り。`名前を` も `名前 を` も同一トークン列に正規化される |
| インデント | タブまたはスペース（同一ファイル内で混在禁止）。`INDENT` / `DEDENT` トークンを生成する |
| `返す` の扱い | `返す` は**キーワードトークン**として出力。識別子末尾からの助詞分離対象にならない |

**助数詞文字の判定規則：**

| 文字種 | 助数詞として認識 | 例 |
|---------|-----|-----|
| 漢字 | ○ | `秒`、`回`、`個`、`行` |
| カタカナ | ○ | `ピクセル`、`バイト` |
| ひらがな | × | 助詞との衝突を回避 |

これにより、`2の` は `数値(2)` + `助詞(の)` に正しく分割され、`5秒` は `数値(5)` + `助数詞(秒)` になる。

**曖昧な全ひらがな識別子の扱い：**

レキサーが助詞分離できない全ひらがな列（例：`できる`、`表示する`）は識別子としてそのまま出力する。助詞分離後の識別子部分がキーワードに完全一致する場合は、キーワードトークンと助詞トークンの組に分割する。それ以外の曖昧な場合は、**空白での明示的分割を要求**する。

### 15.2 パーサで決めるもの

| 責務 | 規則 |
|------|------|
| 助詞式の構造 | `(式 役割助詞)+ 動詞` のパターンマッチでノードを構築。`の` と `は` は助詞式の引数マーカーとしては**使用しない** |
| `の` の結合 | 左結合で連鎖（`aのbのc` = `(aのb)のc`）。属性アクセスまたはスコープ参照として解釈 |
| 算術糖衣構文 | `aとbの和/差/積` パターンを**固定の構文糖衣**として認識。このパターン内の `と` は共同助詞、`の` は構文の一部であり属性アクセスではない |
| `返す` の処理 | `返す` は文キーワードとしてのみ処理。`xを 返す` → `Return(x)`。動詞解決の対象には**ならない** |
| 論理演算子の優先順位 | `かつ` は `または` より優先度が高い（AND > OR） |
| て形チェイン | て形動詞 + `、` の繰り返しでチェインを認識 |
| `分岐して` | て形チェインの途中に挿入される分岐ブロックとして認識 |
| パーサ戦略 | Earley パーサ＋SPPF（初期実装は再帰下降パーサで着手） |

### 15.3 名前解決で決めるもの

| 責務 | 規則 |
|------|------|
| 動詞名の照合 | パーサが「動詞位置にある識別子」として渡し、スコープ内の手順定義と照合 |
| 助詞パターンマッチング | 呼び出し側の助詞セットと定義側の助詞セットを比較し、**完全一致**を要求。余剰・不足はエラー（て形の暗黙 `を` は補完） |
| オーバーロード解決 | 同名手順が複数ある場合、助詞パターンが一意に一致するものを選択。複数候補が残った場合はコンパイルエラー |
| こそあど参照の解決 | §16 の厳密規則に基づいて解決 |
| スコープ探索順序 | 内側から外側へ。`こう` は現在実行中の手順自身 |

### 15.4 型・意味解析で弾くもの

| 責務 | 規則 |
|------|------|
| 助数詞の互換性 | `3回` と `5秒` の加算は型エラー |
| パターンマッチの網羅性 | `どれでもない場合` がない場合は**警告**（エラーではない） |
| 分岐の型整合性 | `分岐して` の各分岐が同じ型を返すことを検証 |
| 未定義変数 | 定義されていない変数の参照はコンパイルエラー |

## 16. スコープ・束縛・こそあどの厳密規則

### 16.1 こそあど参照のスコープ規則

| 指示語 | 参照範囲 | 詳細 |
|--------|----------|------|
| `これ` | 現在のブロック内で最後に評価された式の値 | て形チェインの直前の結果など |
| `それ` | 親ブロック（1段）の環境オブジェクトへの参照 | `それの変数名` で親スコープの変数にアクセス |
| `あれ` | モジュールレベル（大域）の束縛のみ | 大域設定値など |
| `こう` | 現在実行中の手順自身（第一級関数値） | 再帰呼び出し |
| `ここ` | 現在のブロックへの参照（将来仕様、デバッグ用） | — |
| `そこ` | 呼び出し元のコンテキスト（将来仕様、デバッグ用） | — |

**`それ` の二つの用法：**

| 用法 | 構文 | 意味 |
|------|------|------|
| 名前付きアクセス | `それの変数名` | 親スコープの変数にアクセスする |
| 単独使用 | `それ` | 親スコープの文脈値（直近の式の結果） |

通常は明示的な `それの変数名` 形式を推奨する。`それ` 単独は親スコープの結果が明確な文脈（例：条件分岐の直後）でのみ使用すること。

### 16.2 ブロック境界のルール

- `これ` はブロック境界を**越えない**
- `それ` は**1段**の親ブロックのみ参照可能。2段以上は名前参照を強制
- `あれ` はモジュールトップレベルのみ。手順内のローカル変数は `あれ` で参照不可
- 手順呼び出し境界: `それ` は手順境界を**越えない**。手順内から呼び出し元の変数を参照不可（クロージャのキャプチャは別機構）

### 16.3 束縛名とこそあどの優先順位

1. 明示的な束縛名が常に優先
2. `これ` / `それ` / `あれ` / `こう` / `ここ` / `そこ` は予約語であり、変数名として使用不可
3. クロージャ内で `それ` が使われた場合: クロージャの**定義位置**の親ブロックを参照（呼び出し時ではなく定義時のスコープ）
4. 名前による直接キャプチャと `それの変数名` は等価である。どちらも定義時のスコープから捕獲する

### 16.4 `それの変数名` の解釈

`それの変数名` は、親スコープの変数へのアクセスとして解釈される。これは通常の属性アクセス（`オブジェクトのプロパティ`）とは異なる。

```
カウンター という手順
　変わる 数 は 0
　増やす という手順
　　それの数を それの数と1の和 に変える   ※ 親スコープの変数「数」にアクセス
　　それの数を 返す
```

ここで `それの数` は「`それ` が指す親スコープ（`カウンター`）の変数 `数`」として解釈される。

**解釈規則：**

| 式 | 解釈 |
|------|------|
| `それの変数名` | 親スコープの変数へのアクセス（スコープ参照） |
| `値の属性名` | 値のプロパティへのアクセス（属性アクセス） |

`これの属性名` は現在のブロックの値の属性アクセス、`あれの変数名` は大域スコープの変数アクセスとして解釈される。

### 16.5 エラー診断

| 状況 | 診断メッセージ |
|------|---------------|
| ブロック先頭で `これ` を使用 | `「これ」は現在のブロックに値がありません` |
| トップレベルで `それ` を使用 | `「それ」で参照できるスコープがありません` |
| 2段以上遡りを `それ` で試行 | `2段以上外のスコープを参照するには名前を使ってください` |
| 大域に該当する束縛がない | `「あれ」はモジュールレベルの束縛のみ参照できます` |

## 17. 手順呼び出しの意味論

### 17.1 助詞のハイブリッド解釈

助詞は2つの段階で異なる役割を果たす。

1. **パース時（構文要素）**: **役割助詞**（`を/が/に/で/から/まで/と/へ/より`）が `式 役割助詞` の組を引数ノードとしてASTに記録するための構文マーカー
2. **名前解決時（意味的照合）**: 呼び出し側の役割助詞セットを、スコープ内の全候補手順の助詞パターンと照合

**注意**: オーバーロード照合で使用されるのは**役割助詞のみ**である。アクセス助詞 `の` と束縛マーカー `は` は助詞パターンに含まれない（§4.1 参照）。

### 17.2 オーバーロード規則

同名手順は助詞パターン（助詞の**集合**）が異なれば共存可能。

```
割る という手順【数:を、除数:で】       ※ 助詞パターン: {を, で}
割る という手順【一覧:を、個数:に】     ※ 助詞パターン: {を, に}
```

解決順序:
1. 助詞セットの完全一致 → 選択
2. 一致なし → コンパイルエラー

型によるオーバーロードは v0.x では非対応（将来の拡張余地を残す）。

### 17.3 省略された引数の扱い

- て形チェインの暗黙 `を` 渡し**のみ**が省略補完の対象
- それ以外の引数省略はコンパイルエラー
- 単独の `表示する` は、て形チェインの後でなければ `を` 引数がないためエラー

### 17.4 動詞名の解決規則

1. 動詞（手順名）はスコープチェインを内側から外側へ探索
2. `こう` は現在実行中の手順自身（再帰）
3. メソッド呼び出し: `対象の動詞する` は `対象` を暗黙の `が` 引数として `動詞する` を呼び出す。組のメソッドが優先
4. 組み込み手順は最外スコープに存在

### 17.5 HIR の中心ノード

名前解決前:
```
Call { callee: Name, args: Vec<(Particle, Expr)>, implicit_wo: Option<Expr> }
```

名前解決後:
```
ResolvedCall { target: ProcId, args: Vec<(ParamIdx, Expr)> }
```

## 18. 値モデルと評価順序

### 18.1 すべてが式（expression-based）

本言語では、すべての構文要素が値を返す。

| 構文 | 返す値 |
|------|--------|
| 束縛文（`xは 5`） | `無` |
| 条件分岐 | 選択された分岐の最後の式の値 |
| 繰り返し | `無` |
| パターンマッチ | マッチした分岐の値 |
| ブロック | 最後の式の値 |
| 手順定義 | `無` |
| `試す` | 成功時は本体の値、失敗時は `失敗した場合` ブロックの値 |

### 18.2 て形チェインの評価順序

1. 左から右へ順次評価
2. 各ステップの戻り値が次のステップの暗黙 `を` 引数になる
3. `分岐して` の各分岐は必ず値を返す必要がある

### 18.3 途中失敗・例外の扱い

- `訴える` で例外が発生すると、最も近い `試す` ブロックまでスタックを巻き戻す
- `試す` がなければプログラム終了（エラーメッセージ出力）
- `無` は有効な値であり、未定義とは異なる。未定義変数の参照はコンパイルエラー

## 19. コア実行モデル

### 19.1 同期実行が基本

v0.x では全ての実行は同期的。`しながら` / `待つ` / `背景で` は使用時に「未実装」エラー。

### 19.2 コンパイルパイプライン

```
ソースコード → [レキサー] → トークン列
           → [パーサ] → AST
           → [名前解決] → HIR（助詞マッチング済み）
           → [コンパイラ] → バイトコード
           → [VM] → 実行
```

将来的には HIR と バイトコードの間に MIR（制御フロー正規化、こそあど解決済み）を挿入する余地を残す。

### 19.3 バイトコード VM の方針

- スタックベース VM
- 基本命令群: `PUSH`, `POP`, `LOAD`, `STORE`, `CALL`, `RET`, `JMP`, `JMP_IF` 等
- 将来の `SUSPEND` / `RESUME` 命令の予約枠を確保
- 手順呼び出しは `CALL` 命令。引数は助詞順ではなくパラメータインデックス順にスタックに積む

### 19.4 数値型の精度と昇格規則

**内部表現：**

- 整数: 任意精度整数（BigInt 相当）
- 小数: IEEE 754 倍精度浮動小数点数（f64）

**昇格規則：**

| 演算 | 結果の型 | 例 |
|------|---------|-----|
| 整数 ± 整数 | 整数 | `3と1の和` → `4` |
| 整数 × 整数 | 整数 | `3と2の積` → `6` |
| 整数 ÷ 整数 | **常に小数** | `1を2で割る` → `0.5` |
| 整数 % 整数 | 整数 | `5を2で割った余り` → `1` |
| 整数 と 小数の混合演算 | **小数（f64）に昇格** | `3と1.5の和` → `4.5` |
| 小数 ÷ 0 | **NaN** | `0.0を0で割る` → `NaN` |
| 整数 ÷ 0 | **実行時エラー** | `1を0で割る` → エラー |

**比較規則：**

| 比較 | 結果 | 理由 |
|------|------|------|
| `1が 1.0と等しい` | `真` | 数学的等価性で比較 |
| `NaNが NaNと等しい` | `偽` | IEEE 754 準拠 |

**余り演算：**

- 整数同士のみ許可。小数の余りはコンパイルエラー
- 将来的に `fmod` 相当の小数余りを導入する余地を残す

### 19.5 REPL 対応の方針

v0.x では対象外だが、バイトコード VM は REPL との相性が良い。文途中状態は許可しない（改行が文終端のため）。`…` 継続行は REPL でも機能する。

## 20. 文法（EBNF概略）

```ebnf
(* === 文レベル === *)
文          = 束縛文 | 再束縛文 | 式文 | 制御文 | 手順定義 | 組定義 ;
束縛文      = 名前 "は" 式 ;
再束縛文    = 名前 "を" 式 "に" "変える" ;
式文        = 式 ;

(* === 式 === *)
式          = て形連鎖 | 単式 ;
て形連鎖    = 単式 (て形動詞 "、" 連鎖要素)* 終止形動詞 ;
連鎖要素    = 単式 | 途中分岐 ;
途中分岐    = "分岐して" ブロック ;
単式        = 論理式 ;
論理式      = 比較式 (("かつ" | "または") 比較式)* ;
比較式      = 助詞式 | リテラル | 参照 | "(" 式 ")" | メソッド呼出 ;
助詞式      = (式 助詞)+ 動詞 ;
メソッド呼出 = 式 "の" 動詞 ;
助詞        = "を" | "が" | "に" | "で" | "から" | "まで" | "と" | "へ" | "より" ;

(* === 参照 === *)
参照        = こそあど語 | 名前 | 名前 ("の" 名前)* ;
こそあど語  = "これ" | "それ" | "あれ" | "どれ" | "ここ" | "そこ" | "こう" ;

(* === 制御 === *)
制御文      = 条件文 | 繰り返し文 | パターン文 ;
条件文      = "もし" 式 "ならば" ブロック
              ("もしくは" 式 "ならば" ブロック)*
              ("そうでなければ" ブロック)? ;
繰り返し文  = 回数繰り返し | 範囲繰り返し | 条件繰り返し | 走査繰り返し ;

(* === 手順定義 === *)
手順定義    = 手順名 "という手順" 引数宣言? ブロック ;
手順名      = 動詞 ;  (* 推奨: 名詞句込み手順名は避ける *)
引数宣言    = "【" 引数 ("、" 引数)* "】" ;
引数        = 名前 ":" 助詞 ;  (* 正式形式 *)

(* === ブロック === *)
ブロック    = INDENT 文+ DEDENT ;

(* === 文区切り === *)
(* 改行 = 文の終端                               *)
(* "。" = 同一行で複数文を区切る                  *)
(* "、" = て形連鎖内の区切りとしてのみ使用        *)
(* "…" = 次の行に継続                             *)
```

## 21. 予約語

以下の語は現在のバージョンでは未実装だが、将来の互換性のために予約されている。使用した場合は「未実装エラー」となる。

| 予約語     | 想定される用途                    |
| ---------- | --------------------------------- |
| `しながら` | 並行処理（2つの処理を同時に実行） |
| `待つ`     | 非同期処理の待機（await相当）     |
| `背景で`   | バックグラウンド実行              |
