# KLP-110 — 内包表記

> 種別: Kotoba Language Proposal (KLP) — Standards Track（言語機能追加）  
> 番号: KLP-110  
> 題名: 内包表記 — 一覧・対応表の宣言的構築  
> 状態: Draft

---

## 要約

一覧・対応表リテラルの中に `各〜の〜` 形式の内包節を書けるようにする。
Python の `[x*2 for x in items if x > 0]` に相当する構文をことば文法に即した形で導入する。

---

## 動機

現状のことばで「一覧の各要素を変換した新しい一覧」を作ろうとすると、
ループと可変一覧への追加が必要になり冗長である。

```kotoba
※ 現状の書き方
変わる 結果は 【】
元一覧 の 各要素で 繰り返す
  結果に 要素と2の積を 追加する
```

内包表記があれば宣言的に書ける：

```kotoba
結果は 【要素と2の積 | 要素 は 元一覧の各】
```

Python で「map/filter/reduce より読みやすい」と言われる構文がこれである。
`【 】`（一覧リテラル）と `｛ ｝`（対応表リテラル）が既にある以上、
内包表記はその自然な拡張として位置づけられる。

---

## 構文

### 2.1 EBNF 差分

```ebnf
(* 既存 list_lit に内包形を追加 *)
list_lit      ::= "【" list_items "】"
               | "【" comprehension "】" ;           (* 追加 *)

(* 既存 map_lit に内包形を追加 *)
map_lit       ::= "｛" map_items "｝"
               | "｛" map_comprehension "｝" ;       (* 追加 *)

(* 一覧内包 *)
comprehension ::= expr "|" comp_clause+ ;
comp_clause   ::= iter_clause | filter_clause ;
iter_clause   ::= identifier "は" expr "の" "各" ;
filter_clause ::= "かつ" expr ;

(* 対応表内包 *)
map_comprehension ::= expr ":" expr "|" comp_clause+ ;
```

### 2.2 キーワード

`各` はすでに口語で使われうる語であるが、ここでは内包表記の専用トークンとして予約する。
`|`（縦棒）は内包の区切り記号として追加する。

| 区分   | 追加する予約語・記号 |
| ------ | -------------------- |
| 内包   | `各`（iter_clause 内）、`\|`（内包区切り）|

## 意味論

### 3.1 一覧内包

```kotoba
【式 | 変数 は コレクションの各】
```

- `コレクション` の各要素を順に `変数` に束縛して `式` を評価する。
- 結果を集めた新しい `一覧<T>` を返す（`T` は `式` の型）。

### 3.2 フィルタ付き内包

```kotoba
【式 | 変数 は コレクションの各 かつ 条件】
```

- `条件` が `真` の要素のみ含める。
- `条件` は `真偽` 型でなければならない（BLN-001 と整合）。

### 3.3 複数イテレータ（ネスト）

```kotoba
【xとyの積 | x は 行の各 かつ y は 列の各】
```

- 左から右へネストしたループと等価。
- 計算量に注意が必要だが、文法上は禁止しない。

### 3.4 対応表内包

```kotoba
｛キー: 値 | ペア は 元対応表の各｝
```

- `元対応表` の各ペアを `ペア` に束縛して `キー: 値` ペアを生成する。
- 重複キーは後勝ちとする（既存 `対応表` の上書き規則に従う）。

### 3.5 型検査規則

| 規範 ID | 内容                                                                       |
| ------- | -------------------------------------------------------------------------- |
| CMP-001 | `コレクション` は反復可能型（`一覧<T>`・`対応表<K,V>`・KLP-115 のジェネレータ）でなければならない |
| CMP-002 | フィルタ条件は `真偽` 型でなければならない                                 |
| CMP-003 | 一覧内包の結果型は `一覧<式の型>` である                                   |
| CMP-004 | 対応表内包の結果型は `対応表<キー型,値型>` である                          |

### 3.6 診断要件

| 診断コード | トリガー    | メッセージ例（日本語）                                             |
| ---------- | ----------- | ------------------------------------------------------------------ |
| DGN-070    | CMP-001 違反 | 「`数` は反復できません。`一覧<数>` を使ってください」             |
| DGN-071    | CMP-002 違反 | 「フィルタ条件は `真偽` 型が必要ですが `数` が来ました」           |

## コード例

### 4.1 基本変換

```kotoba
数一覧は 【1、2、3、4、5】
二倍一覧は 【要素と2の積 | 要素 は 数一覧の各】
※ 二倍一覧 は 【2、4、6、8、10】
```

### 4.2 フィルタ

```kotoba
偶数一覧は 【要素 | 要素 は 数一覧の各 かつ 要素を2で割った余り が 0 と 等しい】
※ 偶数一覧 は 【2、4】
```

### 4.3 変換＋フィルタ

```kotoba
大きな二倍は 【要素と2の積 | 要素 は 数一覧の各 かつ 要素 が 3 より 大きい】
※ 大きな二倍は 【8、10】
```

### 4.4 対応表内包

```kotoba
点数表は ｛「太郎」: 90、「花子」: 85、「次郎」: 70｝
合格者表は ｛名前: 点 | 名前と点 は 点数表の各 かつ 点 が 80 以上｝
※ 合格者表 は ｛「太郎」: 90、「花子」: 85｝
```
