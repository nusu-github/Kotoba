# KLP-114 — デフォルト引数・可変長引数

> 種別: Kotoba Language Proposal (KLP) — Standards Track（言語機能追加）  
> 番号: KLP-114  
> 題名: デフォルト引数・可変長引数 — 省略可能な仮引数と可変受け取り  
> 状態: Draft  
> 注記: 本文書は言語仕様（`docs/仕様/`）への提案である。`[規範]` ラベルは含まず、

---

## 要約

手順の仮引数に**デフォルト値**（省略時の既定値）と**可変長受け取り**（複数引数の一覧化）を追加する。
Python の `def f(x, y=0)` と `def f(*args)` に相当する機能をことばの助詞体系と整合した形で導入する。

---

## 動機

Python の「小さい道具がサクッと書ける」感覚は、デフォルト引数に大きく依存している。
例えば `print(value, end="\n")` は `end` を省略できるから短く書ける。

ことばは助詞で引数役割を明示するため、「省略＝曖昧」を避けながら入れやすい。
「`を` 助詞を省略したとき既定値を使う」という規則は曖昧さを生まない。

また、「要素を複数渡す」操作（`【1、2、3】` の一覧を直接並べて渡す）を
より自然に書けるようにする可変長引数も合わせて導入する。

---

## 構文

### 2.1 EBNF 差分

```ebnf
(* 仮引数に既定値を追加 *)
param         ::= identifier ":" role_particle ("〔" type_expr "〕")? ("=" expr)? ;

(* 可変長仮引数 *)
variadic_param ::= "〜" identifier ":" role_particle ("〔" type_expr "〕")? ;
param_decl    ::= "【" (param ("、" param)*)? ("、" variadic_param)? "】" ;

(* 可変長引数渡し（展開演算子）*)
particle_arg  ::= access_expr role_particle
               | "〜" access_expr role_particle ;  (* 展開渡し *)
```

### 2.2 予約語の追加

| 区分       | 追加する予約語・記号 |
| ---------- | -------------------- |
| デフォルト | `=`（既定値指定）    |
| 可変長     | `〜`（可変長マーカー）|

`=` は束縛文の `は` とは異なり、仮引数の中でのみ使われる既定値指定子。

## 意味論

### 3.1 デフォルト引数

```kotoba
挨拶する という 手順 【名前:を、丁寧度:に = 「ふつう」】
  もし 丁寧度 が 「丁寧」と 等しい ならば
    「〇〇様、ごきげんよう」
  そうでなければ
    「〇〇さん、こんにちは」
```

- `丁寧度:に` が省略された場合、`「ふつう」` が使われる。
- 省略は助詞で判断される。呼び出し時に `に` 助詞の引数がなければ省略とみなす。

```kotoba
挨拶するを 「太郎」で 呼ぶ               ※ 丁寧度は「ふつう」
挨拶するを 「太郎」で 「丁寧」に 呼ぶ   ※ 丁寧度は「丁寧」
```

### 3.2 デフォルト値の評価タイミング

デフォルト値式は**定義時に一度だけ評価**される（Python のデフォルト引数評価と同じ）。
可変コレクションをデフォルトにする場合の注意点として診断を出すことを推奨する（DGN-112）。

### 3.3 可変長引数

`〜名前:を` は「`を` 役割の引数を 0 個以上受け取り、`一覧<T>` として束縛する」。

```kotoba
合計する という 手順 【〜数値:を〔数〕】 → 数
  ※ 数値 は 一覧<数> として束縛される
  合計は 0
  数値の各 で 繰り返す
    合計を 合計と それのX の 和に 変える
  合計
```

呼び出し：

```kotoba
合計するを 1で 2で 3で 4で 呼ぶ    ※ 数値 は 【1、2、3、4】
合計するを 呼ぶ                      ※ 数値 は 【】（空）
```

### 3.4 展開渡し（スプレッド）

既存の一覧を可変長引数に展開して渡せる：

```kotoba
数一覧は 【1、2、3、4】
合計するを 〜数一覧で 呼ぶ    ※ 数一覧 を展開して渡す
```

### 3.5 型検査規則

| 規範 ID | 内容                                                                          |
| ------- | ----------------------------------------------------------------------------- |
| DEF-001 | デフォルト値の型は仮引数の型（注釈がある場合）と一致しなければならない        |
| DEF-002 | デフォルト引数は必須引数の後ろに置かなければならない                          |
| DEF-003 | 可変長引数は仮引数リストの最後に置かなければならない                          |
| DEF-004 | 同一助詞の可変長引数は手順内に一つしか定義できない                            |
| DEF-005 | 展開渡し（`〜`）の対象は `一覧<T>` 型でなければならない                       |

### 3.6 診断要件

| 診断コード | トリガー    | メッセージ例（日本語）                                                   |
| ---------- | ----------- | ------------------------------------------------------------------------ |
| DGN-110    | DEF-001 違反 | 「`丁寧度` のデフォルト値は `文字列` ですが、型注釈は `数` です」        |
| DGN-111    | DEF-002 違反 | 「デフォルト引数 `丁寧度` は必須引数 `名前` より前に置けません」         |
| DGN-112    | 警告        | 「デフォルト値に可変な `一覧` を使っています。定義時に一度だけ評価されます」 |

## コード例

### 4.1 デフォルト引数

```kotoba
冪乗する という 手順 【底:を〔数〕、指数:に〔数〕 = 2】 → 数
  ※ 実装略（底の指数乗を返す）

冪乗するを 3で 呼ぶ          ※ 3² = 9
冪乗するを 3で 3に 呼ぶ      ※ 3³ = 27
```

### 4.2 可変長引数

```kotoba
最大値を求める という 手順 【〜候補:を〔数〕】 → 数
  ※ 候補 は 一覧<数>
  【候補から 最大のものを 取るを 呼ぶ】   ※ 標準ライブラリ想定

最大値を求めるを 3で 1で 4で 1で 5で 呼ぶ    ※ → 5
```
