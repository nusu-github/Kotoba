# KLP-113 — `表明する`（assert 相当）

> 種別: Kotoba Language Proposal (KLP) — Standards Track（言語機能追加）  
> 番号: KLP-113  
> 題名: `表明する` — 実行時表明（assert）の導入  
> 状態: Draft  
> 注記: 本文書は言語仕様（`docs/仕様/`）への提案である。`[規範]` ラベルは含まず、
>       採択後に対応する仕様ファイルへ規範として移記される。

---

## 要約

`表明する` キーワードを追加し、実行時に前提条件を宣言的に確認できる軽量 assert 構文を導入する。
リリースビルド（`--リリース`）ではコンパイル時に除去できる設計とする。

---

## 動機

ことばには `docs/テスト/適合テスト台帳_v1.md` で示されるようなテスト文化がある。
言語側にも軽量な表明機構があると：

- ライブラリの前提条件ドキュメント（「この引数は正でなければならない」）が実行可能になる
- テストケースを `表明する` で書ける
- デバッグ時に内部不変条件を確認できる

`訴える` は「回復不能なエラー」（ERR-001）に使う。
`表明する` は「正しいプログラムなら到達しないはずの状態」を検査するための、
より軽量で宣言的な機構として位置づける。

---

## 構文

### 2.1 EBNF 差分

```ebnf
(* statement に表明文を追加 *)
statement     ::= ... | assert_stmt ;

assert_stmt   ::= expr "を" "表明する" ("、" string_lit)? ;
```

### 2.2 予約語の追加

| 区分   | 追加する予約語 |
| ------ | -------------- |
| 表明   | `表明する`     |

---

## 意味論

### 3.1 基本動作

```kotoba
条件 を 表明する
条件 を 表明する、「メッセージ」
```

- `条件` が `真` のとき：何も起きない。実行を継続する。
- `条件` が `偽` のとき：表明失敗として `訴える` 相当の例外を発生させる。
  - メッセージが指定されている場合はそのメッセージを付与する。
  - 指定がない場合は「表明が失敗しました」という診断を含むデフォルトメッセージ。

### 3.2 リリースビルドでの除去

`--リリース` フラグを渡してビルドした場合、`表明する` 文はコード生成時に完全に除去される。

このため、`表明する` 式は**副作用を持つ式を条件に含めてはならない**（ASS-002）。
副作用が条件に入っていると、リリースビルドと開発ビルドで挙動が変わるバグの原因になる。

### 3.3 条件の型

`表明する` の条件は `真偽` 型でなければならない（BLN-001 と整合）。

### 3.4 型検査規則

| 規範 ID | 内容                                                                       |
| ------- | -------------------------------------------------------------------------- |
| ASS-001 | `表明する` の条件部は `真偽` 型でなければコンパイルエラー                  |
| ASS-002 | `表明する` の条件部に副作用のある式を含めることを警告（SHOULD NOT）        |
| ASS-003 | `--リリース` ビルドでは `表明する` 文をコード生成段階で除去する            |
| ASS-004 | 表明失敗時の例外は `試す/失敗した場合` で捕捉できる                        |

### 3.5 診断要件

| 診断コード | トリガー    | メッセージ例（日本語）                                            |
| ---------- | ----------- | ----------------------------------------------------------------- |
| DGN-100    | ASS-001 違反 | 「`表明する` の条件は `真偽` 型が必要ですが `数` が来ました」     |
| DGN-101    | ASS-002 警告 | 「`表明する` の条件に副作用があります。リリースビルドで除去されると挙動が変わります」 |
| DGN-102    | 実行時失敗   | 「表明が失敗しました: カウンターは正でなければなりません」        |

## コード例

### 4.1 前提条件の確認

```kotoba
安全に割る という 手順 【左:を〔数〕、右:を〔数〕】 → 数
  右 が 0 と 等しくない を 表明する、「除数はゼロであってはなりません」
  左を 右で 割る
```

### 4.2 テストとしての使用

```kotoba
二倍する という 手順 【値:を〔数〕】 → 数
  値と2の積

二倍するを 5で 呼ぶ が 10 と 等しい を 表明する
二倍するを 0で 呼ぶ が 0 と 等しい を 表明する
```

### 4.3 例外として捕捉

```kotoba
試す
  xが 0 と 等しい を 表明する、「xはゼロのはず」
失敗した場合
  「表明が失敗した」を 表示する
```
