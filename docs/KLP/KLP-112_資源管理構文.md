# KLP-112 — 資源管理構文（`使う間`）

> 種別: Kotoba Language Proposal (KLP) — Standards Track（言語機能追加）  
> 番号: KLP-112  
> 題名: 資源管理構文 `使う間` — `試す/必ず行う` の糖衣  
> 状態: Draft  
> 注記: 本文書は言語仕様（`docs/仕様/`）への提案である。`[規範]` ラベルは含まず、
>       採択後に対応する仕様ファイルへ規範として移記される。

---

## 要約

`使う間` キーワードを導入し、資源（ファイル・ロック・接続等）を自動的に解放する
Python の `with` 文相当の糖衣構文を追加する。

ことばには既に `試す … 必ず行う` がある。`使う間` はこの仕組みの上に乗る
**新しい機能ではなく、読みやすさのための別名構文**である。

---

## 動機

ファイル操作やロック取得のような資源を使うとき、現状のことばでは：

```kotoba
ファイルは 「data.txt」を 開く
試す
  内容は ファイルから 読む
  内容を 表示する
必ず行う
  ファイルを 閉じる
```

と書く必要がある。`使う間` を使えば：

```kotoba
「data.txt」を 開くを 使う間 ファイルとして
  内容は ファイルから 読む
  内容を 表示する
```

と書ける。「資源を使い終わったら必ず閉じる」という意図が一行で読める。

`with` 相当は **Python で「資源管理の定番」**であり、ことばにあることで
ファイル・ロック・一時ディレクトリ・計測等が一気に書きやすくなる。

---

## 構文

### 2.1 EBNF 差分

```ebnf
(* statement に使う間文を追加 *)
statement     ::= ... | with_stmt ;

with_stmt     ::= expr "を" "使う間" (identifier "として")? block ;
```

### 2.2 予約語の追加

| 区分     | 追加する予約語 |
| -------- | -------------- |
| 資源管理 | `使う間`、`として` |

`使う間` は `使う`（use_stmt）とは別のトークンとして「使う間」全体で最長一致する。
`として` は文脈依存の助詞的語であり、`使う間` 構文内でのみ予約語として機能する。

---

## 意味論

### 3.1 脱糖規則

```kotoba
expr を 使う間 名前として
  本体
```

は以下に展開される：

```kotoba
名前は expr
試す
  本体
必ず行う
  名前を 後始末する
```

`後始末する` は、資源を解放するために呼ばれる特性メソッドである（3.2 節参照）。

### 3.2 後始末可能特性（`後始末できる`）

`使う間` に渡せる資源は、`後始末できる` 特性を実装している型のみ。

```kotoba
後始末できる という 特性
  後始末する という 手順
```

標準ライブラリのファイル・ネットワーク接続等はすべてこの特性を実装する。
ユーザーが独自資源を定義する場合も `後始末できる` を `持つ` ことで `使う間` に渡せる。

### 3.3 名前束縛の省略

`として 名前` を省略した場合、資源は名前なしで管理される（本体ブロック内から参照不可）。

```kotoba
ロックを 使う間
  ※ ロックを取得・解放するが、本体内でロック変数を参照しない場合
  共有データを 更新する
```

### 3.4 型検査規則

| 規範 ID | 内容                                                                       |
| ------- | -------------------------------------------------------------------------- |
| RES-001 | `使う間` に渡す式の型は `後始末できる` を実装していなければコンパイルエラー|
| RES-002 | `後始末する` は `使う間` ブロックの正常終了・例外時どちらでも必ず呼ばれる  |
| RES-003 | `後始末する` 内で例外が発生した場合は元の例外を上書きしない（EXC-005 準拠）|
| RES-004 | `名前として` の束縛は `使う間` ブロック内でのみ有効なスコープを持つ        |

### 3.5 診断要件

| 診断コード | トリガー    | メッセージ例（日本語）                                                         |
| ---------- | ----------- | ------------------------------------------------------------------------------ |
| DGN-090    | RES-001 違反 | 「`数` は `後始末できる` を実装していないため `使う間` に渡せません」          |

## コード例

### 4.1 ファイルの読み書き

```kotoba
「output.txt」を 作るを 使う間 ファイルとして
  ファイルに 「一行目」を 書く
  ファイルに 「二行目」を 書く
※ ブロック終了後、ファイルは自動的に閉じられる
```

### 4.2 ユーザー定義資源

```kotoba
タイマー という 組
  開始時刻は 数

タイマー は 後始末できるを 持つ
  後始末する という 手順
    「経過時間: 」と 現在時刻と それのX の 開始時刻 の 差を 表示する

現在時刻を タイマーを 作る 【開始時刻: 現在時刻】を 使う間 計測として
  重い処理を 実行する
※ ブロック終了時に経過時間が表示される
```

### 4.3 名前なし

```kotoba
ロックを 取るを 使う間
  共有カウンターを カウンターと1の和に 変える
```
