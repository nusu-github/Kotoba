> 状態: Draft

## 要約

一覧・文字列の部分取得（スライス）と、整数範囲を表す**範囲型**を追加する。
Python の `a[1:5]`・`range(1, 10)` に相当する操作をことば文法に即した形で導入する。

---

## 動機

現状のことばでは `N番目` で単一要素のアクセスができる。しかし：

- 「3番目から7番目まで」の部分列を取得する構文がない
- 「1から10まで」の整数列を生成する構文がない
- ループで `N回 繰り返す` を書くとき上限値の表現が不自然

この欠如は「配列処理・文字列処理・数値ループ」という日常的な操作で
毎回手動ループを書かせる原因になる。範囲型とスライスが入ると配列処理の快適さが跳ね上がる。

---

## 構文

### 2.1 EBNF 差分

```ebnf
(* アクセス後置子に範囲アクセスを追加 *)
access_tail   ::= identifier
               | integer_lit "番目"
               | "長さ"
               | range_access ;                          (* 追加 *)

range_access  ::= range_expr "番目まで"
               | range_expr "番目から" range_expr "番目まで"
               | range_expr "番目から" ;

(* 範囲リテラル（一次式として追加）*)
primary_expr  ::= ... | range_lit ;
range_lit     ::= expr "から" expr "まで"                (* 両端含む、閉区間 *)
               | expr "から" expr "まで" "（含まず）"   (* 右端を含まない、半開区間 *) ;

(* type_expr に範囲型を追加 *)
type_expr     ::= ... | "範囲" "<" type_expr ">" ;
```

### 2.2 予約語の追加

| 区分 | 追加する予約語                                               |
| ---- | ------------------------------------------------------------ |
| 範囲 | `から`（既存の使用と衝突しないこと確認要）、`まで`、`含まず` |

`から` は `use_stmt`（`「モジュール」から 〜を 使う`）でもすでに使われている。
スライス構文では `数式 から 数式 まで` のトークン列で判別できるため、
同一字句で問題ない（文脈依存で解釈を分ける）。

`まで` と `含まず` は新規予約語として追加する。

## 意味論

### 3.1 範囲型

`範囲<数>` は整数の連続した区間を表す型である。
閉区間（両端含む）と半開区間（右端を含まない）の 2 種類がある。

```kotoba
閉区間は 1から10まで             ※ 1,2,...,10（10を含む）
半開区間は 1から10まで（含まず）  ※ 1,2,...,9（10を含まない）
```

`範囲<数>` は反復可能型（CMP-001 の対象）として扱われ、
`| 変数 は 範囲の各` や `繰り返す 変数 は 範囲の各 ならば` で使える。

### 3.2 スライス（部分列取得）

```kotoba
一覧のN番目からM番目まで
```

- `N` から `M` まで（両端を含む）の部分列を返す。
- 返り値の型は元の型と同じ（`一覧<T>` → `一覧<T>`、`文字列` → `文字列`）。

```kotoba
文字列のN番目から
```

- `N` 番目以降の末尾まで。

### 3.3 負のインデックス

負の整数インデックスは末尾からの逆算とする。

- `-1番目` は最後の要素。
- `-3番目から-1番目まで` は後ろから 3 個。

### 3.4 型検査規則

| 規範 ID | 内容                                                             |
| ------- | ---------------------------------------------------------------- |
| SLC-001 | `一覧` と `文字列` のみスライスを許可する                        |
| SLC-002 | スライスのインデックスは `数` 型（整数）でなければならない       |
| SLC-003 | 範囲リテラルの両端は `数` 型（整数）でなければならない           |
| SLC-004 | 範囲外インデックスは実行時エラーとする                           |
| SLC-005 | `範囲<T>` は `一覧` と `文字列` の他に `対応表` には適用できない |

### 3.5 診断要件

| 診断コード | トリガー     | メッセージ例（日本語）                                 |
| ---------- | ------------ | ------------------------------------------------------ |
| DGN-080    | SLC-001 違反 | 「`対応表` はスライスできません」                      |
| DGN-081    | SLC-002 違反 | 「スライスのインデックスには整数が必要です」           |
| DGN-082    | SLC-004      | 「インデックス `15` が一覧の長さ `10` を超えています」 |

## コード例

### 4.1 基本スライス

```kotoba
文字一覧は 【「a」、「b」、「c」、「d」、「e」】
前半は 文字一覧の1番目から3番目まで
※ 前半 は 【「a」、「b」、「c」】

後半は 文字一覧の4番目から
※ 後半 は 【「d」、「e」】
```

### 4.2 文字列スライス

```kotoba
メッセージは 「こんにちは世界」
先頭は メッセージの1番目から5番目まで
※ 先頭 は 「こんにちは」
```

### 4.3 範囲型でループ

```kotoba
1から10まで（含まず）の各 で 繰り返す
  それのX を 表示する
※ 1,2,...,9 を表示
```

### 4.4 内包表記（KLP-110）との組み合わせ

```kotoba
二乗一覧は 【iとiの積 | i は 1から5までの各】
※ 二乗一覧 は 【1、4、9、16、25】
```
