## 要約

`産出する` キーワードを導入し、**遅延評価のイテレータ**（ジェネレータ）を手順で定義できるようにする。
Python の `yield` に相当。`産出する` を含む手順は呼び出し時に即座に実行されず、
値を一つずつ要求するたびに処理が進む遅延シーケンスを返す。

---

## 動機

全要素を一度にメモリに載せるのではなく「必要なときだけ次の要素を計算する」遅延処理は、
大量データのパイプライン処理・無限シーケンス・I/O ストリームで本質的に有効である。

ことばの `て形チェイン` は「処理の連鎖」を表すのに優れており、
ジェネレータと組み合わせると**遅延パイプライン**が自然に書ける：

```kotoba
無限整数から 偶数だけ取り出して 最初の10件 を 表示する
```

非同期より実装コストが低く、現代的な言語としての説得力を即座に増す機能である。

## 構文

### 2.1 EBNF 差分

```ebnf
(* statement に産出文を追加 *)
statement     ::= ... | yield_stmt ;

yield_stmt    ::= expr "を" "産出する" ;

(* proc_def はジェネレータ手順を自動判別 *)
(* 本体に yield_stmt を含む proc_def は自動的にジェネレータ手順になる *)
```

### 2.2 予約語の追加

| 区分         | 追加する予約語 |
| ------------ | -------------- |
| ジェネレータ | `産出する`     |

## 意味論

### 3.1 ジェネレータ手順の識別

本体に `産出する` 文を含む手順定義は**ジェネレータ手順**として扱われる。
ジェネレータ手順を呼び出すと、実行は開始されずに `反復可能<T>` 型の**ジェネレータオブジェクト**が返る。

```kotoba
無限カウンターを数える という 手順
  変わる n は 0
  繰り返す
    nを 産出する
    nを nと1の和に 変える
```

上記の呼び出し：

```kotoba
カウンターは 無限カウンターを数えるを 呼ぶ
※ カウンター は 反復可能<数> 型
```

### 3.2 ジェネレータの進行

ジェネレータオブジェクトが反復（`各`、内包表記 CMP-001 の対象）されるたびに：

1. 最後に止まった `産出する` 文の直後から実行を再開する。
2. 次の `産出する` 文に到達した時点で、産出する値を返して一時停止する。
3. 手順の末尾（または `返す` 文）に達したとき、ジェネレータは完了する。

### 3.3 `産出する` と `返す` の共存

- `産出する` と `返す` を同一手順内で使える。
- `返す` はジェネレータを**終了させる**（産出する値なしで完了）。
- 戻り型注釈（`→`）はジェネレータ手順に付けない。戻り型は `反復可能<T>` として自動推論される（`T` は産出する式の型）。

### 3.4 て形チェインとの連携

ジェネレータは `反復可能<T>` を実装しているため、
内包表記（KLP-110）・スライス（KLP-111）・手順の可変長引数（KLP-114）と組み合わせられる。

```kotoba
偶数のみ という 手順 【元:を〔反復可能<数>〕】
  元の各 で 繰り返す
    もし それのX を 2で 割った余り が 0 と 等しい ならば
      それのXを 産出する

無限カウンターを数えるを 呼ぶを 偶数のみで して 最初の10個を 取って 表示する
```

### 3.5 型検査規則

| 規範 ID | 内容                                                                     |
| ------- | ------------------------------------------------------------------------ |
| YLD-001 | `産出する` は手順の本体内でのみ使用できる                                |
| YLD-002 | `産出する` を含む手順の戻り型は `反復可能<T>` として推論される           |
| YLD-003 | ジェネレータ手順に戻り型注釈を付ける場合、`反復可能<T>` でなければエラー |
| YLD-004 | `産出する` する値の型がすべて同じでなければコンパイルエラー（型統一）    |
| YLD-005 | `反復可能<T>` は内包表記・スライス・可変長引数の反復対象として使用できる |

### 3.6 診断要件

| 診断コード | トリガー     | メッセージ例（日本語）                                  |
| ---------- | ------------ | ------------------------------------------------------- |
| DGN-120    | YLD-001 違反 | 「`産出する` は手順の外では使えません」                 |
| DGN-121    | YLD-004 違反 | 「`産出する` の値の型が一致しません: `数` と `文字列`」 |

## コード例

### 4.1 有限ジェネレータ

```kotoba
Nまでの数を返す という 手順 【上限:を〔数〕】
  変わる i は 1
  繰り返す i が 上限 以下 ならば
    iを 産出する
    iを iと1の和に 変える

Nまでの数を返すを 5で 呼ぶの 各で 繰り返す
  それのXを 表示する
※ 1,2,3,4,5 を順に表示
```

### 4.2 無限ジェネレータ＋内包表記（KLP-110）

```kotoba
フィボナッチ数列 という 手順
  変わる a は 0
  変わる b は 1
  繰り返す
    aを 産出する
    変わる 次は aとbの和
    aを bに 変える
    bを 次に 変える

最初の10個は 【x | x は フィボナッチ数列を呼ぶの各】の 1番目から10番目まで
最初の10個を 表示する
```

### 4.3 パイプライン処理

```kotoba
偶数のみ取り出す という 手順 【元:を〔反復可能<数>〕】
  元の各 で 繰り返す
    もし それのX を 2で 割った余り が 0 と 等しい ならば
      それのXを 産出する

二倍にする という 手順 【元:を〔反復可能<数>〕】
  元の各 で 繰り返す
    それのXと2の積を 産出する

1から100まで（含まず）を
  偶数のみ取り出すで して
  二倍にするで して
  最初の5個を 取り出して
  表示する
```
