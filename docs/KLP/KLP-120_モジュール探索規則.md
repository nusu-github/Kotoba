# KLP-120 — モジュール探索規則

> 種別: Kotoba Language Proposal (KLP) — Standards Track（言語機能追加）  
> 番号: KLP-120  
> 題名: モジュール探索規則 — 検索パス・相対参照・キャッシュ  
> 状態: Draft  
> 注記: 本文書は言語仕様（`docs/仕様/`）への提案である。`[規範]` ラベルは含まず、
>       採択後に対応する仕様ファイルへ規範として移記される。

---

## 要約

v1 の `「モジュール」を 使う` 構文（MOD-001〜005）は構文としては定義されているが、
「どこからモジュールを探すか」という**探索規則**が未確定である。

本提案は、Python の `sys.path` 相当のモジュール探索順序・相対インポート・
ロードキャッシュの規則を規範として確定させる。

---

## 動機

探索規則がないと「`使う` を書いてもどこのファイルを読むか実装依存」という状態になる。
これは複数の実装が出たとき互換性の崩壊原因になり、
パッケージ管理（KLP-121）の基盤にもなるため早めに確定させる必要がある。

---

## モジュール探索順序

### 2.1 探索パスの優先順位

`「名前」を 使う` が評価されるとき、以下の順序で探索する：

1. **組み込みモジュール**：言語処理系が内蔵する標準 I/O・数値操作等
2. **現在のファイルと同じディレクトリ**
3. **`KOTOBA_PATH` 環境変数**（`:` 区切りの追加パス）
4. **標準ライブラリパス**（処理系のインストール先）
5. **`使う場所.toml` のパッケージ依存パス**（KLP-121 が採択された場合）

最初に見つかったモジュールを使用する。見つからない場合はコンパイルエラー（MOD-011）。

### 2.2 相対参照

モジュール名の先頭に `./` を付けると、**現在のファイルからの相対パス**として解決する。

```kotoba
「./utils」を 使う
「../common/helpers」を 使う
```

`/` は KLP-120 の文脈でのみパス区切りとして解釈される。
モジュール名が `./` または `../` で始まる場合に限り相対参照として扱い、
そうでない場合は絶対探索（2.1 の順序）とする。

### 2.3 ファイル名マッピング

| `使う` に書く名前 | 探索するファイル           |
| ----------------- | -------------------------- |
| `「utils」`       | `utils.kb`                 |
| `「math/vector」` | `math/vector.kb`           |
| `「./helpers」`   | `（現在ファイルと同じ dir）/helpers.kb` |

拡張子 `.kb` は自動補完する（明示指定も可）。

## ロードキャッシュ

### 3.1 同一モジュールの重複ロード防止

**[規範 MOD-010]** 同一の絶対パスに解決されるモジュールは、
一度ロードされた後はキャッシュから返す。重複実行はしない。

これにより「A と B が同じ C を `使う` しても C の初期化は一度だけ」が保証される。

### 3.2 循環参照の検出

**[規範 MOD-011]** モジュールのロード中に自分自身（または循環）を再び `使う` しようとした場合、
コンパイルエラーとする。

| 診断コード | メッセージ例（日本語）                                       |
| ---------- | ------------------------------------------------------------ |
| DGN-130    | 「モジュール `utils` が見つかりません（探索パス: …）」      |
| DGN-131    | 「循環参照: `a` → `b` → `a`」                               |

## 型検査規則

| 規範 ID | 内容                                                                        |
| ------- | --------------------------------------------------------------------------- |
| MOD-010 | 同一絶対パスのモジュールはキャッシュから返し、重複実行しない                |
| MOD-011 | モジュールが見つからない場合はコンパイルエラー                              |
| MOD-012 | 循環参照はコンパイルエラー                                                  |
| MOD-013 | `公開` のない定義は外部から `使う` できない（MOD-003 の再確認）             |
| MOD-014 | `相対参照`（`./`・`../`）は探索パス 1 番の組み込みモジュールより優先しない  |
