# KLP-103 — 条件式の型制約

> 種別: Kotoba Language Proposal (KLP) — Standards Track（言語機能追加）  
> 番号: KLP-103  
> 題名: 条件式の型制約 — `もし` / `繰り返す` 条件は `真偽` のみ  
> 状態: Draft  
> 注記: 本文書は言語仕様（`docs/仕様/`）への提案である。`[規範]` ラベルは含まず、
>       採択後に対応する仕様ファイルへ規範として移記される。

---

## 要約

`もし`・`もしくは`・`繰り返す`（while）の条件式の型を **`真偽` のみ**に制限する。
Python の truthy（空リスト・0・空文字列が偽に扱われる）を禁止し、
条件が必ず `真偽` 型であることを静的に保証する規則を追加する。

---

## 動機

Python は `if x:` と書けば `x` がゼロや空コレクションのとき偽に扱われる。
便利だが、「数値が条件に渡った」「None が条件に渡った」といったバグが静的に検出できない。

ことば v1 の設計思想は「裏側は一意決定」（GEN-001〜003）であり、
「同一語に複数の意味を与えない」（GEN-003）が中心にある。
条件式の truthy は「真偽ではない値が真偽として振る舞う」という多義性そのものであり、
この設計方針と相容れない。

条件を `真偽` に限定することで：

- 「`数` や `文字列` を誤って条件に渡す」バグを静的に排除できる
- `もし 値 が 0 と 等しくない ならば` のように明示的に書く文化が生まれる
- 仕様が締まり、型検査と診断が単純になる

## 意味論

### 2.1 型制約規則

| 規範 ID | 内容                                                                         |
| ------- | ---------------------------------------------------------------------------- |
| BLN-001 | `もし` 式の条件部（`expr`）の型は `真偽` でなければコンパイルエラー         |
| BLN-002 | `もしくは` の各条件部の型は `真偽` でなければコンパイルエラー               |
| BLN-003 | `繰り返す` 条件式の型は `真偽` でなければコンパイルエラー                   |
| BLN-004 | `真` と `偽` はリテラルとして `真偽` 型を返す（既存の意味論を明文化）       |
| BLN-005 | 比較式（`より大きい`・`等しい` 等）は `真偽` 型を返す（既存の意味論を明文化）|
| BLN-006 | 論理式（`かつ`・`または`・`でない`）のオペランドは `真偽` 型でなければエラー |
| BLN-007 | `真偽` 型への暗黙変換（truthy 変換）を禁止する                              |

### 2.2 `繰り返す` 構文の追加

v1 の `loop_expr` はカウンタ・条件なしの無限ループのみをサポートしている（`抜ける` で脱出）。
`繰り返す 条件 ならば ブロック` 形式の条件付きループ（while ループ）を追加する。

```ebnf
(* 既存 loop_expr に条件ループを追加 *)
loop_expr     ::= "繰り返す" block                    (* 無限ループ（既存）*)
               | "繰り返す" expr "ならば" block ;      (* 条件ループ（追加）*)
```

### 2.3 診断要件

| 診断コード | トリガー    | メッセージ例（日本語）                                              |
| ---------- | ----------- | ------------------------------------------------------------------- |
| DGN-030    | BLN-001 違反 | 「`もし` の条件は `真偽` 型が必要ですが `数` が渡されました」       |
| DGN-031    | BLN-003 違反 | 「`繰り返す` の条件は `真偽` 型が必要ですが `文字列` が渡されました」|
| DGN-032    | BLN-007 違反 | 「`数` を `真偽` として扱うことはできません。`が 0 と 等しくない` のように明示してください」 |

診断 DGN-032 はヒント（`.with_hint`）として具体的な書き方を添える。

## コード例

### 3.1 正しい条件式（受理される）

```kotoba
もし カウンター が 0 より 大きい ならば
  カウンターを 表示する

もし 完了 ならば        ※ 完了 は 真偽 型の束縛
  「終わった」を 表示する

繰り返す カウンター が 10 より 小さい ならば
  カウンターを 表示する
  カウンターを カウンターと1の和に 変える
```

### 3.2 誤った条件式（拒否される）

```kotoba
もし カウンター ならば    ※ DGN-030: カウンターは数型
  ※ ...

もし 一覧 ならば          ※ DGN-030: 一覧は一覧型（空かどうかの確認は 長さが0と等しい で書く）
  ※ ...
```

### 3.3 Python 的な書き方の言への変換

| Python                  | ことば                                            |
| ----------------------- | ------------------------------------------------- |
| `if x:`                 | `もし xは 空でない ならば`（長さ != 0 の判定）    |
| `if x is not None:`     | `もし x が 無 と 等しくない ならば`               |
| `if count:`             | `もし カウンター が 0 と 等しくない ならば`       |
| `while items:`          | `繰り返す 項目一覧の長さ が 0 より 大きい ならば` |
