## 1. 差分方針

- 互換維持より、機械的な一意解釈を優先する。
- 日本語らしさは 4 本柱（助詞呼出・て形チェイン・こそあど参照・助数詞型）で保持する。
- 仕様のみ更新し、実装追従は次フェーズで行う。

---

## 2. ルール三分割（規範候補/廃止候補/保留候補）

### 2.1 規範候補（v1 で採用）

- 助詞三分類（役割助詞/アクセス助詞/束縛マーカー）
- 助詞集合による呼び出し解決
- て形チェインと `分岐して`
- 助数詞型の静的検査
- if/match/try を式として扱う値モデル
- `試す` の値規則（finally 値破棄を含む）
- 予約語管理（将来予約語を含む）
- 標準 I/O API（`表示する` / `入力する` / `読む` / `書く`）の規範化

### 2.2 廃止候補（v1 で廃止）

- `これ/それ/あれ` の単独使用
- 無名仮引数 `【:を】`
- メソッド呼び出し時の暗黙 `が` 注入
- 概略 EBNF だけで運用する方針

### 2.3 保留候補（v1 では規範化しない）

- 並行処理（`しながら/待つ/背景で`）
- マクロ機構
- パッケージ管理
- REPL 実装詳細

## 3. 破壊的変更一覧（理由付き）

| ID | v0.4 | v1 | 互換破壊理由 | 影響 |
| --- | --- | --- | --- | --- |
| BC-001 | `これ` 単独を許可 | `これのX` のみ許可 | 文脈値と束縛値の衝突排除 | 単独 `これ` コードは修正が必要 |
| BC-002 | `それ` 単独を許可 | `それのX` のみ許可 | 親文脈値と親環境参照の二義性排除 | 単独 `それ` を全置換 |
| BC-003 | `あれ` 単独を許可 | `あれのX` のみ許可 | 大域値解釈の曖昧性排除 | 単独 `あれ` を全置換 |
| BC-004 | 無名仮引数 `【:を】` 許可 | 禁止 | 署名同値判定の不安定性を解消 | 仮引数名の明示が必要 |
| BC-005 | `返す` の文法定義が不完全 | 文法・意味で完全定義 | 実装依存挙動を排除 | 仕様解釈差が消える |
| BC-006 | `試す` の文法定義が不完全 | 文法・意味で完全定義 | 実装との差異を解消 | try 構文の解析が安定 |
| BC-007 | `使う` の文法規定が断片的 | `use_stmt` を正式化 | モジュール文の構文差異解消 | import 文エラーが明確化 |
| BC-008 | `公開` の構文上の位置が曖昧 | `public_decl` で限定 | 公開対象の誤適用防止 | 無効 `公開` がエラー化 |
| BC-009 | メソッド暗黙 `が` 許可 | 禁止 | 暗黙補完を一本化 | 呼出しで助詞明示が必要 |
| BC-010 | 省略補完の境界が曖昧 | て形暗黙 `を` のみ許可 | 省略ルールの決定性確保 | て形外省略コードがエラー化 |
| BC-011 | EBNF が概略で未定義項目あり | 完全正式文法 | 仕様空白をゼロ化 | 既存の曖昧解釈が不可 |
| BC-012 | `、` 用途が説明中心 | 連鎖区切り専用を規範化 | 文分割の誤解釈排除 | 非連鎖 `、` がエラー化 |
| BC-013 | 助詞付着分離の境界が曖昧 | 最長一致を必須化 | 字句解釈の一意化 | 一部トークン分割結果が変更 |
| BC-014 | 予約語運用が部分的 | 一覧を完全規範化 | 実装間の予約語差異を排除 | 予約語利用コードが失敗 |
| BC-015 | 型解決への拡張余地を記述 | 型解決オーバーロードを禁止 | 呼出し解決の予測可能性確保 | 型だけで呼び分け不可 |
| BC-016 | `しながら/待つ/背景で` の扱いが散在 | 未実装予約語として統一 | 将来互換を維持しつつ現行誤用防止 | 使用時エラーが固定 |
| BC-017 | if/match/try の式規則が分散 | 一章で統合 | 値モデルの矛盾排除 | 条件式の戻り値解釈が固定 |
| BC-018 | ループ値規則が説明中心 | `無` を規範化 | 実装差分を防止 | ループ戻り値依存コード修正 |
| BC-019 | `必ず行う` の例外上書きが注意書き中心 | 規範化 | 例外伝搬結果を固定 | finally 内再送出の結果が固定 |
| BC-020 | 助数詞型を概念説明中心 | 次元型として規範化 | 型検査可能性を向上 | 暗黙変換依存コードが失敗 |
| BC-021 | 診断要件が例示中心 | 必須フィールド化 | ツール間診断差を縮小 | 診断実装更新が必要 |
| BC-022 | `公開` 対象外への適用時挙動が曖昧 | 明示エラー化 | 静的検査の確実性向上 | 既存の寛容実装と非互換 |
| BC-023 | `それ` の遡り規則が説明依存 | 1 段親に固定 | 捕捉範囲の予測可能性向上 | 深い遡り記法は修正必要 |
| BC-024 | `return/try/use/public` の本文と文法不一致 | 一致必須 | 仕様内矛盾を除去 | 旧解釈は無効 |
| BC-025 | 入力 API 名が `入力を受ける` | 入力 API 名を `入力する` に統一 | 束縛 `は` との読みやすさ・一貫性を優先 | 旧 API 名の置換が必要 |

## 6. 移行ガイド（仕様読者向け）

1. 単独 `これ/それ/あれ` を全検索し、`の` 付き参照へ変換する。
2. 無名仮引数をすべて命名仮引数へ置換する。
3. て形以外の暗黙引数呼び出しを明示助詞呼び出しへ置換する。
4. `公開` の適用先を定義文に限定する。
5. 差分 ID（BC-001..025）単位でコードベース影響を見積もる。
6. `入力を受ける` を `入力する` へ置換する。
