> 状態: Draft
> 方針: 仕様先行・一貫性優先
> 対象: 仕様文書のみ（実装追従は別フェーズ）

## 1. 仕様範囲・用語・規範記法

### 1.1 適用範囲

**[規範] GEN-001 (MUST)**
本仕様は、言（ことば）言語の字句・構文・意味・型・実行モデル・診断・適合性を定義する。

**[規範] GEN-002 (MUST)**
本仕様における曖昧な自然言語表現は、すべて本章の用語定義と規範キーワードに従って解釈する。

**[参考]**
v1 は v0.4 を凍結参照しつつ、互換より一貫性を優先して再定義する。

### 1.2 用語定義

- `役割助詞`: 手順呼び出しの引数役割を示す助詞（`を/が/に/で/から/まで/と/へ/より`）
- `アクセス助詞`: 属性・環境参照のための助詞（`の`）
- `束縛マーカー`: 束縛を表す助詞（`は`）
- `助詞式`: `(式 役割助詞)+ 動詞` で表す呼び出し式
- `アクセス式`: `式 の 識別子` の左結合連鎖
- `束縛式`: `束縛対象 は 式`（文）
- `て形チェイン`: 暗黙 `を` 受け渡しを伴う処理連鎖
- `環境参照`: `これのX/それのX/あれのX` によるスコープ参照

**[規範] GEN-003 (MUST)**
同一語に複数の意味を与えてはならない。意味が複数候補になる場合は、構文で分岐させる。

### 1.3 規範キーワード

**[規範] GEN-004 (MUST)**
`MUST`、`MUST NOT`、`SHOULD`、`SHOULD NOT`、`MAY` を以下の意味で用いる。

| キーワード | 意味 |
| --- | --- |
| MUST | 必須要件 |
| MUST NOT | 禁止要件 |
| SHOULD | 推奨要件 |
| SHOULD NOT | 非推奨要件 |
| MAY | 任意要件 |

### 1.4 記述ラベル

**[規範] GEN-005 (MUST)**
各節は `[規範]` と `[参考]` を明示する。`[規範]` のみが適合判定対象となる。

## 2. 字句仕様

### 2.1 トークンカテゴリ

**[規範] LEX-001 (MUST)**
字句解析器は次のカテゴリを区別する。

- リテラル（整数、小数、文字列、真偽、無）
- 識別子
- 助詞（役割助詞、アクセス助詞、束縛マーカー）
- キーワード
- 記号（`【】｛｝（）`、`、`、`。`、`:`、`…`）
- 構造トークン（`INDENT`、`DEDENT`、`SEP`、`EOF`）

### 2.2 文区切り・インデント

**[規範] LEX-002 (MUST)**
改行と `。` は等価な文区切りとして `SEP` に正規化する。

**[規範] LEX-003 (MUST)**
同一ファイル内でタブとスペースの混在インデントを禁止する。

**[規範] LEX-004 (MUST)**
`、` はて形チェイン内部の区切りにのみ使用する。

### 2.3 助詞分離

**[規範] LEX-005 (MUST)**
識別子末尾に助詞が付着した場合、最長一致で分離する。

例:

- `名前を` -> `識別子(名前)` + `助詞(を)`
- `地点まで` -> `識別子(地点)` + `助詞(まで)`
- `比較より` -> `識別子(比較)` + `助詞(より)`

**[規範] LEX-006 (MUST NOT)**
`の` と `は` は助詞式の引数助詞として扱ってはならない。

### 2.4 キーワード優先

**[規範] LEX-007 (MUST)**
キーワード完全一致は助詞分離より優先する。

例:

- `そうでなければ` は単一キーワード
- `どれかを調べる` は単一キーワード

### 2.5 数値と助数詞

**[規範] LEX-008 (MUST)**
`数値 + 助数詞` は単一の助数詞付き数値として字句化する。

**[規範] LEX-009 (MUST)**
助数詞の字種は漢字・カタカナのみを許可し、ひらがなを禁止する。

例:

- `5秒` は有効
- `100ピクセル` は有効
- `2の` は `整数(2)` + `アクセス助詞(の)`

### 2.6 予約語

**[規範] LEX-010 (MUST)**
以下は予約語であり、識別子として使用してはならない。

| 区分 | 予約語 |
| --- | --- |
| 制御 | `もし` `ならば` `もしくは` `そうでなければ` `次へ` `抜ける` |
| 例外 | `試す` `失敗した場合` `必ず行う` `訴える` |
| 定義 | `という` `手順` `組` `特性` `公開` |
| 束縛 | `は` `変わる` `変える` `返す` |
| モジュール | `使う` |
| パターン | `どれかを調べる` `の場合` `どれでもない場合` `どれでもない` `どれか` |
| 連鎖 | `分岐して` |
| 論理 | `かつ` `または` `でない` |
| こそあど | `これ` `それ` `あれ` `こう` `ここ` `そこ` |
| 将来予約 | `しながら` `待つ` `背景で` |

**[規範] LEX-011 (MUST)**
`しながら`、`待つ`、`背景で` は v1 では未実装予約語とし、使用時は未実装エラーとする。

### 2.7 字句エラー

**[規範] LEX-012 (MUST)**
字句エラーは行・列・スパンとともに報告し、以降の処理を継続可能な最短回復を行う。

## 3. 構文仕様（完全正式文法）

### 3.1 構文方針

**[規範] SYN-001 (MUST)**
`助詞式`、`束縛式`、`アクセス式` は文法規則を分離して定義する。

**[規範] SYN-002 (MUST)**
`返す`、`試す`、`使う`、`公開` は文法上の独立規則を持つ。

### 3.2 EBNF

```ebnf
program            ::= sep* statement (sep+ statement)* sep* EOF ;
sep                ::= NEWLINE | "。" ;

statement          ::= public_decl
                     | mutable_bind_stmt
                     | bind_stmt
                     | rebind_stmt
                     | return_stmt
                     | continue_stmt
                     | break_stmt
                     | proc_def
                     | struct_def
                     | trait_def
                     | trait_impl
                     | use_stmt
                     | expr_stmt ;

public_decl        ::= "公開" (proc_def | struct_def | trait_def) ;

mutable_bind_stmt  ::= "変わる" bind_target "は" expr ;
bind_stmt          ::= bind_target "は" expr ;
bind_target        ::= identifier | destructure_pattern ;
rebind_stmt        ::= identifier "を" expr "に" "変える" ;

return_stmt        ::= "返す" expr? ;
continue_stmt      ::= "次へ" ;
break_stmt         ::= "抜ける" ;

proc_def           ::= identifier "という" "手順" param_decl? block ;
param_decl         ::= "【" param ("、" param)* "】" ;
param              ::= identifier ":" role_particle ;

struct_def         ::= identifier "という" "組" block ;
trait_def          ::= identifier "という" "特性" block ;
trait_impl         ::= identifier "は" identifier "を" "持つ" block ;

use_stmt           ::= string_lit "を" "使う"
                     | string_lit "から" import_items "を" "使う" ;
import_items       ::= string_lit ("、" string_lit)* ;

expr_stmt          ::= expr ;

expr               ::= te_chain_expr | logical_or_expr ;

te_chain_expr      ::= te_head "、" te_tail ;
te_head            ::= te_step_te ;
te_tail            ::= te_segment ("、" te_segment)* ;
te_segment         ::= te_step_te | te_step_stop | te_branch ;
te_step_te         ::= chain_arg* verb_te ;
te_step_stop       ::= chain_arg* verb_stop ;
chain_arg          ::= expr role_particle_non_wo ;
te_branch          ::= "分岐して" block ;

logical_or_expr    ::= logical_and_expr ("または" logical_and_expr)* ;
logical_and_expr   ::= unary_not_expr ("かつ" unary_not_expr)* ;
unary_not_expr     ::= comparison_expr ("でない")? ;

comparison_expr    ::= additive_expr
                     | additive_expr "が" additive_expr "より" "大きい"
                     | additive_expr "が" additive_expr "より" "小さい"
                     | additive_expr "が" additive_expr "以上"
                     | additive_expr "が" additive_expr "以下"
                     | additive_expr "が" additive_expr "と" "等しい"
                     | additive_expr "が" additive_expr "と" "等しくない" ;

additive_expr      ::= multiplicative_expr
                     | multiplicative_expr "と" multiplicative_expr "の" "和"
                     | multiplicative_expr "と" multiplicative_expr "の" "差" ;

multiplicative_expr ::= postfix_expr
                      | postfix_expr "と" postfix_expr "の" "積"
                      | postfix_expr "を" postfix_expr "で" "割る"
                      | postfix_expr "を" postfix_expr "で" "割った余り" ;

postfix_expr       ::= particle_call_expr | access_expr ;
particle_call_expr ::= particle_arg+ verb_stop ;
particle_arg       ::= access_expr role_particle ;

access_expr        ::= primary_expr ("の" access_tail)* ;
access_tail        ::= identifier
                     | integer_lit "番目"
                     | "長さ" ;

primary_expr       ::= literal
                     | identifier
                     | kosoado_ref
                     | "こう"
                     | list_lit
                     | map_lit
                     | lambda_expr
                     | if_expr
                     | match_expr
                     | loop_expr
                     | try_expr
                     | "(" expr ")" ;

kosoado_ref        ::= ("これ" | "それ" | "あれ") "の" identifier ;

if_expr            ::= "もし" expr "ならば" block
                       ("もしくは" expr "ならば" block)*
                       ("そうでなければ" block)? ;

match_expr         ::= expr "は" "どれかを調べる" match_block ;
match_block        ::= INDENT match_arm+ DEDENT ;
match_arm          ::= pattern "の場合" block
                     | "どれでもない場合" block ;

pattern            ::= literal
                     | identifier
                     | "どれか"
                     | "【" pattern ("、" pattern)* "】" ;

loop_expr          ::= times_loop_expr
                     | range_loop_expr
                     | while_loop_expr
                     | foreach_loop_expr ;

times_loop_expr    ::= expr "回" "繰り返す" loop_var? block ;
range_loop_expr    ::= expr "から" expr "まで" "繰り返す" loop_var? block ;
while_loop_expr    ::= expr "間" "繰り返す" block ;
foreach_loop_expr  ::= expr "の" "それぞれについて" loop_var block ;
loop_var           ::= "【" identifier "】" ;

try_expr           ::= "試す" block catch_clause? finally_clause? ;
catch_clause       ::= "失敗した場合" catch_param? block ;
catch_param        ::= "【" identifier ":" "で" "】" ;
finally_clause     ::= "必ず行う" block ;

lambda_expr        ::= "（" param_decl? lambda_body "）" ;
lambda_body        ::= statement (sep+ statement)* ;

list_lit           ::= "【" (expr ("、" expr)*)? "】" ;
map_lit            ::= "｛" (map_entry ("、" map_entry)*)? "｝" ;
map_entry          ::= identifier ":" expr ;

destructure_pattern ::= list_destructure | map_destructure ;
list_destructure   ::= "【" identifier "、" identifier "…" "】" ;
map_destructure    ::= "｛" identifier ("、" identifier)* "｝" ;

block              ::= INDENT statement (sep+ statement)* sep* DEDENT ;

literal            ::= integer_lit | float_lit | string_lit | bool_lit | none_lit | counter_lit ;
counter_lit        ::= (integer_lit | float_lit) counter_word ;

role_particle      ::= "を" | "が" | "に" | "で" | "から" | "まで" | "と" | "へ" | "より" ;
role_particle_non_wo ::= "が" | "に" | "で" | "から" | "まで" | "と" | "へ" | "より" ;

verb_stop          ::= identifier ;
verb_te            ::= identifier_te ;
```

### 3.3 構文制約

**[規範] SYN-003 (MUST NOT)**
`【:を】` のような無名仮引数宣言を禁止する。仮引数は必ず `名前:助詞` 形式とする。

**[規範] SYN-004 (MUST NOT)**
`返す` を手順名にしてはならない。

**[規範] SYN-005 (MUST)**
`公開` は `手順`・`組`・`特性` の定義にのみ適用できる。

**[規範] SYN-006 (MUST)**
`かつ` は `または` より高優先順位とする。

## 4. 名前束縛とスコープ

### 4.1 束縛モデル

**[規範] SCP-001 (MUST)**
変数は既定で不変束縛とし、`変わる` で可変束縛を宣言する。

**[規範] SCP-002 (MUST)**
再束縛は `名前を 式に 変える` のみ許可する。

### 4.2 こそあど参照

**[規範] SCP-003 (MUST)**
`これ`、`それ`、`あれ` は単独使用を禁止し、`の` を伴う明示参照のみ許可する。

**[規範] SCP-004 (MUST)**
`それのX` は 1 段外の親環境のみ参照可能とする。

**[規範] SCP-005 (MUST)**
`あれのX` はモジュールレベル束縛のみ参照可能とする。

**[規範] SCP-006 (MUST)**
`こう` は現在実行中の手順値を指す再帰参照とする。

### 4.3 スコープ優先順位

**[規範] SCP-007 (MUST)**
名前解決は内側スコープから外側スコープの順に探索する。

**[規範] SCP-008 (MUST NOT)**
予約語を束縛名として使用してはならない。

## 5. 呼び出し意味論

### 5.1 助詞マッチング

**[規範] CAL-001 (MUST)**
手順呼び出しは、呼び出し側の助詞集合と定義側の助詞集合の完全一致で解決する。

**[規範] CAL-002 (MUST NOT)**
型情報によるオーバーロード解決を行ってはならない。

**[規範] CAL-003 (MUST NOT)**
同一定義内で同一助詞を複数仮引数に割り当ててはならない。

### 5.2 省略規則

**[規範] CAL-004 (MUST)**
省略引数補完は、て形チェインの暗黙 `を` のみ許可する。

**[規範] CAL-005 (MUST NOT)**
メソッド呼び出しにおける暗黙 `が` 注入を行ってはならない。

**[規範] CAL-006 (MUST NOT)**
て形チェイン以外で必須助詞引数を省略してはならない。

### 5.3 `返す` の意味

**[規範] CAL-007 (MUST)**
`返す` は文キーワードであり、`返す` という手順呼び出しとして解釈してはならない。

## 6. て形チェイン

### 6.1 基本規則

**[規範] CHN-001 (MUST)**
て形チェインは左から右へ評価し、直前ステップの結果を次ステップの暗黙 `を` 引数に渡す。

**[規範] CHN-002 (MUST)**
て形ステップには、明示引数として `を` 以外の役割助詞引数のみ指定できる。

**[規範] CHN-003 (MUST)**
チェイン全体は最終終止形ステップの値を返す。

### 6.2 分岐して

**[規範] CHN-004 (MUST)**
`分岐して` ブロックの各分岐は値を返さなければならない。

**[規範] CHN-005 (MUST)**
`分岐して` の結果値は次ステップの暗黙 `を` 引数に接続する。

### 6.3 失敗伝搬

**[規範] CHN-006 (MUST)**
チェイン中に例外が発生した場合、最寄りの `試す` まで伝搬する。

## 7. 型システム

### 7.1 型方針

**[規範] TYP-001 (MUST)**
型システムは静的型推論中心で動作する。

**[規範] TYP-002 (MUST)**
型推論不能箇所はコンパイルエラーとする。

### 7.2 基本型

基本型:

- `数`
- `文字列`
- `真偽`
- `一覧<T>`
- `対応表<K, V>`
- `手順`
- `無`

### 7.3 助数詞型（次元型）

**[規範] TYP-003 (MUST)**
助数詞付き数値は `数<次元>` として扱う。

**[規範] TYP-004 (MUST NOT)**
次元の異なる値同士の四則演算を許可してはならない。

**[規範] TYP-005 (MUST)**
同一次元の加減算は同一次元を返す。

### 7.4 数値昇格

**[規範] TYP-006 (MUST)**
整数は任意精度整数、小数は IEEE 754 倍精度とする。

**[規範] TYP-007 (MUST)**
`整数 ÷ 整数` の結果は小数とする。

**[規範] TYP-008 (MUST)**
整数と小数の混合演算は小数に昇格する。

**[規範] TYP-009 (MUST)**
`整数 ÷ 0` は実行時エラー、`小数 ÷ 0` は IEEE 754 に従う。

### 7.5 比較規則

**[規範] TYP-010 (MUST)**
`1` と `1.0` は等しいと判定する。

**[規範] TYP-011 (MUST)**
`NaN` は `NaN` と等しくない。

### 7.6 準形式規則（抜粋）

```
Γ ⊢ e1 : 数<d>    Γ ⊢ e2 : 数<d>
----------------------------------  (T-AddDim)
Γ ⊢ e1 + e2 : 数<d>

Γ ⊢ e1 : 数<d1>   Γ ⊢ e2 : 数<d2>   d1 ≠ d2
---------------------------------------------  (T-AddDimMismatch)
Γ ⊢ e1 + e2 : 型エラー

Γ ⊢ e1 : 数<整数> Γ ⊢ e2 : 数<整数>
------------------------------------  (T-DivInt)
Γ ⊢ e1 / e2 : 数<小数>
```

## 8. 制御構造（式意味論）

### 8.1 if 式

**[規範] CTR-001 (MUST)**
`もし` 式は選択された分岐ブロックの最終値を返す。

**[規範] CTR-002 (MUST)**
`もしくは` は上から順に評価し、最初に真となる分岐のみ実行する。

### 8.2 match 式

**[規範] CTR-003 (MUST)**
`どれかを調べる` は上から順に最初に一致するアームを実行する。

**[規範] CTR-004 (SHOULD)**
`どれでもない場合` を記述することを推奨する。

### 8.3 loop 式

**[規範] CTR-005 (MUST)**
繰り返し式の値は `無` とする。

**[規範] CTR-006 (MUST)**
`次へ` と `抜ける` はループ文脈内でのみ使用できる。

## 9. 例外処理

### 9.1 構造

**[規範] EXC-001 (MUST)**
`試す` は式である。成功時は本体値、失敗時は `失敗した場合` 値を返す。

**[規範] EXC-002 (MUST)**
`失敗した場合` がないときに例外が起きた場合、`試す` 式の値は `無` とする。

**[規範] EXC-003 (MUST)**
`必ず行う` の評価結果は常に破棄する。

### 9.2 例外伝搬

**[規範] EXC-004 (MUST)**
`訴える` は最寄りの `試す` へ例外を送出する。

**[規範] EXC-005 (MUST)**
`必ず行う` 内で再送出した例外は元例外を上書きする。

**[規範] EXC-006 (SHOULD NOT)**
`必ず行う` 内での再送出は避けるべきである。

## 10. モジュール

### 10.1 読み込み

**[規範] MOD-001 (MUST)**
`「モジュール」を 使う` はモジュール全体を導入する。

**[規範] MOD-002 (MUST)**
`「モジュール」から 「項目」を 使う` は明示項目のみ導入する。

### 10.2 公開

**[規範] MOD-003 (MUST)**
定義は既定で非公開とする。

**[規範] MOD-004 (MUST)**
`公開` 指定された手順・組・特性のみ外部公開される。

**[規範] MOD-005 (MUST NOT)**
`公開` を式文や束縛文に適用してはならない。

## 11. 実行モデル

### 11.1 評価順序

**[規範] RUN-001 (MUST)**
式評価は規則で定義された順序で決定的に行う。

**[規範] RUN-002 (MUST)**
同一入力・同一外部状態では同一結果を返す。

### 11.2 非同期予約語

**[規範] RUN-003 (MUST)**
`しながら`、`待つ`、`背景で` の評価は常に未実装エラーとする。

## 12. 診断規則

### 12.1 診断の必須情報

**[規範] DGN-001 (MUST)**
診断は `種別/位置/原因/修正指針` を含む。

### 12.2 主要診断

**[規範] DGN-002 (MUST)**
